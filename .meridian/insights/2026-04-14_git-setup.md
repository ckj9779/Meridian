### INS-009 | failure_and_fix | Step 4
**Observation:** GPG signing from WSL when invoked via `wsl.exe -e bash -c '...'` (non-interactive) failed with `Inappropriate ioctl for device` because pinentry needs a TTY. Resolution: warm up gpg-agent with an interactive sign in the user's WSL terminal (`echo test | gpg --clearsign`) so the passphrase is cached (default 600s TTL); subsequent non-TTY commits succeed within the cache window.
**Affected:** Any automated/scripted commit flow; future CI-like signing.
**Proposed action:** Document this workflow in `docs/GITOPS.md`. Long-term alternatives: (a) raise gpg-agent cache TTL (`default-cache-ttl 7200` in `~/.gnupg/gpg-agent.conf`), (b) configure a TTY-less pinentry (pinentry-gtk via WSLg), (c) for CI runners, use `--pinentry-mode loopback` with a secret-manager-supplied passphrase.
**Severity:** warning

### INS-010 | failure_and_fix | Step 4
**Observation:** `git push` from WSL hung / errored on `git-credential-manager` because: (1) WSL git had `credential.helper=manager` globally (set by earlier Windows-side `git-credential-manager` install), and (2) `/usr/bin/gh` in WSL is actually **gitsome** (unrelated tool), not GitHub CLI, so `gh auth setup-git` wouldn't have worked either. Resolution: push from Git Bash on Windows, which has a working `gh` CLI and credential manager. The already-signed commit is just an object in `.git/` — any authenticated git binary can push it; signing and pushing are separable.
**Affected:** All future pushes from WSL to GitHub.
**Proposed action:** Options to consider: (a) install the real GitHub CLI in WSL (`apt remove gitsome && install gh` from the official repo), (b) set up SSH-based remote (`git remote set-url origin git@github.com:ckj9779/Meridian.git`), (c) configure WSL git to use Windows credential manager via interop: `git config --global credential.helper '/mnt/c/Program\ Files/Git/mingw64/bin/git-credential-manager.exe'`, or (d) adopt the split pattern — sign in WSL, push from Git Bash. Pick one and document in GITOPS.md.
**Severity:** warning

### INS-008 | failure_and_fix | Step 3c
**Observation:** `ln -sf ../../.meridian/hooks/check-headers.sh .git/hooks/pre-commit` from WSL failed with "Operation not permitted" on the Windows NTFS `D:\` mount. Classic WSL+NTFS symlink limitation — works on ext4 but not on Windows-mounted filesystems without special flags.
**Affected:** Pre-commit hook installation approach.
**Proposed action:** Used `git config core.hooksPath .meridian/hooks` instead. This is **local config** (only set on this clone). Follow-ups for future: (a) document this setup step in `docs/GITOPS.md` "Cloning the repo" section, (b) consider committing a `scripts/setup-hooks.sh` that new clones run once, or (c) evaluate `husky` once Node.js is adopted.
**Severity:** info
