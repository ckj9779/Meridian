## Insights

### INS-001 — Claude Code runs in Git Bash, not WSL
- **Type:** environment
- **Observation:** Claude Code operates in Git Bash on Windows (`/d/Meridian`) not WSL (`/mnt/d/Meridian/`). The GPG signing key `799AD4A789D27DA8` lives in WSL's keyring only. Git had zero signing config (no `user.signingkey`, `commit.gpgsign`, or `gpg.program`).
- **Context:** Pre-flight checks at sprint start — `gpg --list-secret-keys` returned empty in Git Bash; `wsl.exe gpg --list-secret-keys` found the key.
- **Routing:** CLAUDE.md Part 2 Environment section (machine description should note the Claude Code environment is Git Bash, not direct WSL), docs/GITOPS.md (GPG signing section should document the bridge pattern)
- **Severity:** high
- **Action taken:** Created `.meridian/gpg-wsl-bridge.sh` wrapper that delegates to `wsl.exe gpg`. Configured local git config: `gpg.program = .meridian/gpg-wsl-bridge.sh`, `user.signingkey = 799AD4A789D27DA8`, `commit.gpgsign = true`. Verified: test commit produced valid GPG signature embedded in commit object. Test commit reset after verification.

### INS-006 — check-headers.sh was never wired as a git pre-commit hook
- **Type:** convention_gap
- **Observation:** `.meridian/hooks/check-headers.sh` exists and is executable, and `core.hooksPath = .meridian/hooks` is set. But git requires hook files to be named after the hook (e.g. `pre-commit`). A file named `check-headers.sh` is never triggered automatically. The header check has likely never blocked or warned on any commit.
- **Context:** Step 7 — examining hooks directory to wire check-secrets.sh into the pre-commit chain.
- **Routing:** docs/GITOPS.md "Hook installation" section (should note the dispatcher pattern)
- **Severity:** medium
- **Action taken:** Created `.meridian/hooks/pre-commit` dispatcher that calls check-secrets.sh (fail-fast) then check-headers.sh. Both hooks now actually execute on every commit attempt.

### INS-005 — GPG-agent restart clears passphrase cache; loopback pinentry required
- **Type:** failure_and_fix
- **Observation:** After applying `gpg-agent.conf` TTL settings and restarting the agent (`gpgconf --kill && --launch`), the passphrase cache was cleared. Subsequent `git commit` failed with "Inappropriate ioctl for device" — the bridge script couldn't prompt for passphrase through a non-TTY pipe.
- **Context:** Step 5 — committing GITOPS.md after applying GPG-agent config.
- **Routing:** docs/GITOPS.md GPG agent section (document `allow-loopback-pinentry` requirement + `--pinentry-mode loopback` in bridge script)
- **Severity:** medium
- **Action taken:** Added `allow-loopback-pinentry` to WSL `~/.gnupg/gpg-agent.conf`. Updated `.meridian/gpg-wsl-bridge.sh` to pass `--pinentry-mode loopback`. Commit succeeded after fix. GITOPS.md already documents the bridge pattern.
